批量下载档案使用说明

批量下载档案的操作流程较为复杂，不仅耗时较长，且服务器与客户端在运算过程中容易出现异常情况。为确保操作顺利进行，请按照本说明操作。

在档案下载过程中，可切换操作界面执行其他任务；关闭软件重新打开，仍可继续执行之前未完成的下载任务。

一、下载前的准备工作

（一）路径设置要求

建议将电脑客户端与档案存储路径设置在同一盘符（如D盘、E盘等），避免使用系统盘（C盘）。如此设置，可方便管理硬盘占用量，也对数据保护有利。

（二）存储空间预留

每份档案（包含输出至档案存储目录的 PDF 文件及电脑客户端中的缓存文件）平均占用空间约16Mb；每户（涵盖在享、停保、在办业务的所有户数）平均对应 3.2 份档案。你需提前根据业务数据规模，计算所需硬盘存储空间，并为目标盘符预留充足容量。由于缓存文件与 PDF 档案文件共同存储，整体占用空间较大，这也是推荐将客户端与档案存放路径置于同一盘符的原因。

（三）360等防护软件误报

360等杀毒防护软件对桌面客户端及其使用的 JAVA 程序提示误报。请在误报时放行，或关闭类似360等防护软件。

（四）客户端下载

若原客户端打开有问题，请按照《民政业务管理信息系统客户端软件分发页》（点击查看 🔍）的指引下载最新客户端。

二、使用方法详解

（一）任务创建与档案存储路径选择

点击软件“下载档案”界面右下角的“创建新的下载任务” 按钮，系统将创建并开启新的下载任务，稍等片刻，系统会弹出档案输出目录选择对话框。此时，请选择与电脑客户端处于同一盘符下的文件夹作为档案存储路径，以保障数据存储、管理和保护的便捷性。

（二）任务进程说明

档案下载分为初始化、数据准备、生成档案三个阶段：

1. 初始化：完成初始化后，系统将提示用户选择档案输出目录（操作方法见前文）。

2. 数据准备：此阶段耗时与数据量大小和客户端缓存直接相关，特别是客户端缓存，若有缓存，则仅需准备缓存后发生了变化的数据，能极大地提高档案生成效率。进入该阶段后，下载任务列表将显示选定的档案存储路径，点击其右侧的文件夹图标，可以打开该路径。

3. 生成档案：此阶段耗时与数据量大小和客户端缓存直接相关，特别是客户端缓存，若有缓存，则仅需准备缓存后发生了变化的数据，能极大地提高档案生成效率。进入此阶段后，任务列表将展示任务描述信息，并通过数字直观呈现下载进度。若存在档案生成失败的情况，任务结束后，用户可点击任务描述右侧的文件夹按钮，打开日志文件夹，将其中的日志文件发送给客服人员，以便核查失败原因。

（三）任务状态解读

档案下载任务包含执行中、已中断、已结束三种状态：

1. 执行中：任务正在正常运行。

2. 已中断：若在任务执行过程中关闭系统，任务状态将变为已中断；任务执行过程中发生了不可逆的错误（如操作系统内存溢出、网络中断等）也会导致任务中断。用户可点击 “已中断” 右侧的播放按钮，可继续执行任务。

3. 已结束：任务处理完毕。若存在档案生成失败的情况，用户可点击任务描述右侧的文件夹按钮，打开日志文件夹，将其中的日志文件发送给客服人员，以便核查失败原因。

（四）后台任务操作

档案下载均为后台任务，即便任务处于执行中状态，用户仍可切换模块执行其他业务操作。当有任务正在执行时，界面右上角会出现带有读取图标的任务执行列表，点击其中的任务条目，可直接跳转至对应档案下载页面。

三、注意事项

（一）本地缓存管理

档案下载任务均为客户端本地任务，任务信息仅存储于客户端，未在服务器留存。若删除客户端程序，所有缓存数据（包括任务进度与相关信息）将一并被删除。因此，用户需谨慎删除客户端程序，避免意外导致数据丢失。

（二）并发任务处理

用户可尝试同时执行多个下载档案任务，但针对同一类业务，并行执行多个任务不仅无法加速下载进程，反而会延长整体耗时（总时长 = 同类任务数 * 单个任务时长）。由于档案生成依赖 java 程序运行，若电脑配置较低（内存小于 16G），不建议同时启动多个下载任务，以免因系统资源不足导致任务卡顿甚至崩溃。

（三）档案输出路径规范

档案输出目录采用层级结构：选定的档案输出目录 -> 县级地区名称（仅县级用户存在此层级） -> 镇级地区名称（县、镇级用户存在此层级） -> 村级地区名称（县、镇、村级用户存在此层级） -> 业务类别名称 -> 档案类别名称（如有） -> 档案类别名称（如有） -> 城乡分类名称（如有） -> 业务状态名称 -> 档案文件。为便于上级向下级分发档案及按地区核对数据，建议每次选择同一档案输出目录，实现业务档案的统一地区归类。当然，用户也可根据管理需要按业务类别分别选择输出目录。

四、数据安全保障

（一）本地缓存内容解析

本地缓存数据包含三类：

1. 预生成档案 PDF 文件：未加密。通过数据关联记录末次变更时间，缓存用于加速档案生成。

2. 预计算业务数据：未加密。源自服务器计算结果。通过数据关联记录末次变更时间，缓存用于加速档案生成。

3. 其它各类文档图片：加密文件。

（二）物理性保护措施

为实现便捷、高效的数据物理保护，建议将电脑客户端与档案存放路径设置在同一盘符。如此一来，数据的物理保护可简化为对该硬盘（对应盘符）的防护。若需多盘备份，可将档案生成目录与客户端软件一同压缩后拷贝至其他物理硬盘，安排专人负责管理，限制外借、规范使用。

（三）未加密数据处理

预生成的档案 PDF 文件和预计算业务数据文件存储于 “客户端安装根目录\cached-data\archives” 目录下。由于该目录文件数量庞大，若出于数据保护目的需删除未加密数据，请勿直接打开目录进行操作（避免因文件过多导致 windows 操作系统内存溢出），应直接删除整个目录。删除后再次下载档案，系统将重新整理全部数据；若保留这些文件，后续仅需处理缓存变更后的数据，可显著提升下载效率。

五、其它

（一）单文件下载

为了方便下载会议材料（如公示照片、民主评议材料照片）等，电脑客户端提供了单张图片（或单个文件）下载功能。在电脑客户端点击图片，可打开图片查看图层，在该图层的右上角，有“关闭”、“放大”、“缩小”、“保存”按钮，点击“保存”按钮，可将图片解密并保存到电脑本地。

（二）单户下载档案

电脑客户端原提供的单户下载档案功能仍可正常使用。
